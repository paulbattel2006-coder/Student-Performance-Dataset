# Datasets für die Gruppenarbeit

Dieses Repository enthält die Datasets für das Gruppenprojekt im 2. Semester des Data Science Kurses. Jeder Gruppe wird ein Dataset zugewiesen, das mit den folgenden Aufgaben bearbeitet werden soll:

* [ ] **Das Dataset-Readme lesen:** Lest die Markdown-Datei, die zu eurem Dataset gehört. Diese Datei befindet sich ebenfalls im Ordner „data“ und enthält einige grundlegende Informationen über das Dataset.
* [ ] **Das Dataset laden:** Falls notwendig, bereinigt das Dataset, um eventuelle Probleme beim Laden zu beheben.
* [ ] **Das Dataset exploriert:** Achtet dabei besonders auf potenzielle Probleme mit den Daten (z. B. fehlerhaft formatierte Datensätze, unplausible Werte, problematische Encodings). Entwickelt falls nötig eine Strategie, um diese Probleme zu lösen.
* [ ] **Eine Machine-Learning-Strategie entwickeln:** Überlegt euch eine potenzielle Idee, wie ihr dieses Projekt mithilfe von Algorithmen für überwachtes/unüberwachtes maschinelles Lernen umsetzen könnt (d. h. welche Features genutzt und welche Zielvariablen modelliert werden können).
* [ ] **Das Dataset bereinigen:** Entwickelt eine Preprocessing-Pipeline, die eure Machine-Learning-Idee unterstützt, und bereitet die Daten so vor, dass sie bereit für das Training sind.
* [ ] **Ein Machine-Learning-Modell erstellen:** Trainiert euer Modell und evaluiert es. Passt falls nötig eure vorherigen Schritte an, um das Modell weiter zu verbessern.
* [ ] **Alles schriftlich festhalten:** Refactort euren Code, dokumentiert eure Erkenntnisse und reicht euer Ergebnis als eine in sich geschlossene ZIP-Datei ein. Diese ZIP-Datei muss die folgenden Dateien enthalten:
  * Die finale Version eurer Datendatei
  * Die finale Version eures Jupyter Notebooks
  * Eure requirements.txt
  * Euren Abschlussbericht (kann auch direkt im Jupyter Notebook enthalten sein)

> **Einsatz von LLMs in der Gruppenarbeit:** Die Nutzung von Large Language Models (LLMs) wie ChatGPT, Gemini und Co. ist für dieses Projekt grundsätzlich erlaubt. Der Einsatz solcher Werkzeuge muss jedoch im Abschlussbericht transparent dokumentiert werden (inklusive der Angabe, wozu und in welchem Umfang sie genutzt wurden). Bitte beachtet: Als Gruppe steht ihr in jedem Fall vollumfänglich für euren eingereichten Code ein. Das bedeutet, dass jedes Gruppenmitglied alle durchgeführten Arbeitsschritte und Programmierungen lückenlos verstehen und erklären können muss.
